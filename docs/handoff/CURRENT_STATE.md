### 리스트 복귀 시 탭/상태 유지 (PR #31)

- 문제: `/list`에서 탭/상태 선택 후 상세 진입 → "목록으로" 복귀 시 초기 조건(활동/가보고 싶은 곳)으로 리셋
- 진단: 탭/상태는 URL query(`?tab=&status=`)에 반영되나 복귀 경로(returnTo)가 카드→상세 경로에만 연결. `buildListReturnTo`가 tab/status를 실어 나르도록 정리
- 구현: 상세 진입 시 현재 리스트 조건을 returnTo로 전달, "목록으로"가 그 경로로 복귀. 없으면 기본 목록 fallback
- **스코프 경계**: 검색어·필터는 URL이 아닌 내부 state(useState)라 이번 스코프에서 제외 → 별도 백로그로 분리

### 신규 등록 후 상세 화면으로 이동 (PR #32)

- 등록하기 후 `/list`가 아닌 방금 등록한 항목 상세(`/activities/[id]`·`/places/[id]`)로 이동
- 진단: 등록 API가 이미 생성 항목 id 반환(`.insert().select().single()`) → API 무수정. Form의 `router.push('/list')`만 상세 경로로 변경
- top loader 프로그래매틱 전환 규칙 준수(전환 직전 start(), 성공 경로 done() 미호출)

### 상태 전환 문구 통일 + 참고 링크 표시/스킴 보정 (PR #33)

- **문구 통일**: 상세 하단 역방향 버튼 "가보고 싶은 곳으로 되돌리기" → "가보고 싶은 곳으로"(카드 메뉴 `STATUS_MENU_LABELS.wishlist`와 통일). 정방향 "다녀왔어요"는 유지

- 패스코드는 **6자리 고정**이다(4~6자리 아님).

- portal 요소는 `.page` 토큰 상속 못 받음(단 `:root` 승격 토큰 `--s-sub`·`--s-faint`는 상속 가능).
